openai 注册 chatgpt

Posted on 2023-10-16 Edited on 2023-10-25

一.准备工作

和注册美区 Apple ID 一样的流程：

  1. 挂个代理，伪装在日本、新加坡或者美国，建议新加坡；亲测香港是 100%不可行的
  2. 准备一个国外手机号，GoogleVoice 虚拟号会被识别，亲测不行，使用接码平台
  3. Chrome 浏览器。

二. 注册接码平台

打开网站：sms-activate.org/ 使用邮箱注册 注意：邮件里链接点击后可能回 404，多试几次就好了。

然后，选择充值 1 美元，支持使用支付宝。（我充值的时候最低 2 美元）

充值完毕，在左侧搜索 OpenAI，从销量看印尼最高，亲测也可用。(印度也可以) 选择后系统会分配一个手机号，留着备用。

三. 注册 OpenAI

打开链接 beta.openai.com/signup 选择使用 Google 登录即可（建议）

在这里输入第二步中分配给你的手机号，然后点击【Send code via SMS】按钮。


注意：
如果直接输入 6282339295075 google 验证失败，可尝试将前缀删除
比如 6282339295075 –> 82339295075 ，如还是不行，申请其他国家的短信验证

后面使用的 阿根廷 ，直接输入 541133679612 验证通过
回到刚刚的接码平台，就能看到收到的验证码了。 PS：注意购买后的短信有效期是 20 分钟，需要快速操作哦~
我们把验证码拷贝出来输入到 OpenAI 的注册界面即可

四. 体验 ChatGPT

重新登录：chat.openai.com/auth/login
然后访问：chat.openai.com/chat
这时你就可以开始尽情和机器人聊天了

参考

  1. https://sms-activate.org/cn/freePrice#activation
  2. ChatGPT 保姆级教程，一分钟学会使用 ChatGPT！

qnap 小米电视共享文件夹

Posted on 2023-10-11 Edited on 2023-10-25
  1. 检查存储池状态是否正常，如果卷报读取/删除，有快照的话删除快照，在重启下机器


| 2. 检查存储池是否正常，如果卷报只读错误，建议备份好文件数据重新做系统

  1. 控制台-网络与文件服务-启用微软网络-工作组为 WORKGROUP-最低 SMB1-访问改为禁用

  2. 如果 NAS 中配置好后还是不行，打开 windows 控制面板-程序-启用或关闭 Windows 功能-勾选 SMB

    检测报什么错误，一般用 windows+R 运行\\ip

  3. 启用不安全的来宾登录（此操作是在电脑组策略中）—Windows+R 运行 gpedit.msc –管理模板-网络-Lanman 工作站-启用不安全的来宾登录

  4. 如果提示不允许一个或者多个用户同时访问，重启电脑或者清除一下电脑的连接就好，Windows+R 运行 cmd 执行 net use /d /y *回车，多运行几次

  5. 如果上述操作做完还是不行，IP 地址是固定的话可以改为动态获取，SMB 去映射新的 IP 地址，控制台-网络与虚拟交换机-网络适配器-设定

  6. 还未解决可以通过 ssh 登录，用 putty 登录上去执行以下命令

  7. #smb2status 查看 smb 服务状态

  8. #cd /share/CACHEDEV1_DATA/ 进入卷目录中

  9. #ll 查看文件

  10. #mv .samba .samba.backup 重命名 SMB

  11. #/etc/init.d/smb.sh restart 重启 SMB 服务


  12. 执行命令后还是不行，可以更换一台电脑，利用 SMB 去连接，不排除是电脑问题

  13. 更换电脑操作后还不行，执行恢复默认的 smb 服务，在上述操作后，执行以下

  14. #cp /etc/default_config/smb.conf /share/CACHEDEV1_DATA /Publi 复制 smb 配置文件

  15. #cp /etc/default_config/smb.conf /mnt/HDA_ROOT/.config/ smb.conf 恢复默认 smb

  16. 执行完成需要重启一下 NAS，在控制台-共享文件夹-创建-手动指定路径-还原共享文件夹

  17. 以上操作完成后，还是连接不了 SMB，建议备份数据，重做系统
